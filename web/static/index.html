Concept Description

Understanding Emergency VoIP

A deep dive into low-bandwidth voice communication using raw UDP sockets, OPUS codec, and adaptive network techniques.

📡
What is VoIP?
Voice over Internet Protocol (VoIP) transmits voice as digital data packets over IP networks instead of traditional phone lines. In emergency scenarios, bandwidth-constrained VoIP enables critical communication when infrastructure is damaged or overloaded.
🔄
How It Works
  • Microphone captures raw PCM audio (8kHz, 16-bit mono)
  • OPUS encoder compresses to 6–20 kbps with DTX + FEC
  • Custom header added: [seq | timestamp | codec | payload]
  • UDP socket transmits packet to receiver
  • Jitter buffer reorders packets, fills gaps
  • OPUS decoder reconstructs audio for playback
⚡
Why UDP Over TCP?
TCP's retransmission delays are fatal for real-time audio. UDP provides:
  • Zero retransmission overhead — no head-of-line blocking
  • Lower latency — no 3-way handshake or ACK waiting
  • Custom reliability — we add our own seq numbers + FEC
  • Better for lossy networks — old packets are useless for voice
🎯
Purpose & Use Cases
Designed for emergency smart city networks where bandwidth is constrained.
  • Disaster zones with damaged cellular infrastructure
  • First responder communication over mesh networks
  • IoT-connected emergency stations in smart cities
  • Low-bandwidth satellite or radio links
📊
Key Parameters
  • Bitrate: 6 / 12 / 20 kbps (adaptive)
  • Sample Rate: 8000 Hz narrowband
  • Frame Size: 20ms (160 samples)
  • Packet Header: 15 bytes (seq + ts + codec + len)
  • Jitter Buffer: 60–200ms adaptive
  • MOS Score: 1.0–5.0 quality metric
🌐
Real-World Systems
This design mirrors production VoIP technology:
  • WebRTC: Browser-based calls (Google Meet, Discord)
  • SIP/RTP: Enterprise telephony protocol stack
  • Zoom/Teams: OPUS codec + adaptive bitrate
  • Amateur Radio: Codec2 over digital radio modes

📝 Step-by-Step Process
1

Audio Capture

Microphone captures raw PCM audio at 8kHz, 16-bit mono — 160 samples per 20ms frame.

2

OPUS Encoding

Raw PCM is compressed by the OPUS codec at the selected bitrate with DTX (silence suppression) and FEC (error correction).

3

Packet Framing

A 15-byte custom header [seq(4B) | timestamp(8B) | codec(1B) | length(2B)] is prepended to the encoded payload.

4

UDP Transmission

Packets are sent via UDP SOCK_DGRAM — no connection setup, no retransmits, minimal overhead.

5

Network Transit (Loss, Delay, Jitter)

Packets traverse the network. Some are lost, others delayed or arrive out of order.

6

Jitter Buffer & Playback

Receiver's jitter buffer reorders packets, fills gaps with silence (PLC), then OPUS decodes to speaker.

❓ Help & Instructions

Getting Started:

1. Both PCs must be on the same WiFi/LAN network.
2. Start the Receiver on PC B first (click Dashboard → Receiver → Start).
3. Start the Sender on PC A with PC B's IP address.
4. Speak into the microphone — audio streams in real time.


Impairment Proxy: Simulates packet loss, delay, and jitter.

Simulation Tab: Visualize how UDP packets travel from sender to receiver.


Troubleshooting:

• Allow UDP port 5004 through your firewall
• Install dependencies: pip install pyaudio opuslib
• Windows: Download opus.dll to the project directory
